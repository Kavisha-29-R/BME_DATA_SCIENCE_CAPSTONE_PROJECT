# Project Submission Requirements

## Submission Guidelines

- Submit your project as file uploads on Canvas.  
  - If there is more than one file, submit a single **zip file** containing all necessary files.

- Your submission must include:
  1. **Code** for a data-driven web app (Streamlit) that you create based on an application of your interest.  
  2. A **5-minute (maximum) video** demonstrating your project.  
     - The video can be included in the repository or linked in your README.  
  3. A **one-page Markdown README** describing your project.  
     - Your README should include:
       - A description of your project
       - Instructions for running the app
       - A link to the demonstration video (or embed if possible)

- **Identification:**  
  - Your name must appear in your README, in the project video, and as comments in all code files.

---

## Grading Rubric

Your project will be evaluated based on the following criteria:

### Mandatory Requirements

1. **Data & Analysis:**  
   - The project must involve data and analysis. This could be prediction, regression, classification, or any other type of data-driven task.  
   - Projects that do not involve data will **not be accepted**.

2. **Streamlit App:**  
   - The project must include a Streamlit web app.  
   - Projects without a Streamlit app will **not be accepted**.

3. **Teamwork:**  
   - Teams may consist of **no more than 5 members**.  
   - Each team member must submit their own README and 5-minute video.  
   - Larger teams will be expected to produce more substantial projects.
   - Or Individually done.

---

### Evaluation Criteria

Projects will be graded on:

- **Documentation:**  
  - Clear instructions on how to use the app.

- **Effort:**  
  - The project should demonstrate significant effort.  
  - Simple or trivial apps (e.g., BMI calculator) may result in point deductions.

- **Correctness:**  
  - Was the data handled appropriately?  
  - Was it cleaned, processed, and analyzed correctly?  
  - Are there any major conceptual or practical errors?

- **Communication:**  
  - Is the app well-presented?  
  - Does the README and video clearly explain what the app does and how to use it?

- **Overall Quality:**  
  - Completeness, usability, and polish of the app and supporting materials.
